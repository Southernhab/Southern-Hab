Rework the entire Southern Habitat Consulting website for the best possible mobile experience.

This task applies to all pages.

PRIMARY GOAL

At screen widths of 768px and below, the site should feel intentionally designed for mobile rather than simply compressed from desktop.

Keep the full desktop version intact.

Do not rewrite approved copy.
Do not change page URLs.
Do not change form destinations.
Do not remove required legal or licensing limitations.
Do not change the desktop layout unless a desktop rule is causing mobile overflow.

MOBILE BREAKPOINTS

Use:

@media (max-width: 768px)

and add a smaller breakpoint for narrow phones:

@media (max-width: 430px)

GLOBAL MOBILE LAYOUT

At 768px and below:

- Set all content to one column
- Remove fixed widths
- Remove min-width values
- Prevent horizontal scrolling
- Use width: 100%
- Use max-width: 100%
- Use box-sizing: border-box
- Use 18px horizontal page padding
- Use 32px to 48px vertical section spacing
- Reduce oversized blank sections
- Remove desktop-only fixed heights
- Keep images responsive
- Stack buttons vertically
- Keep all touch targets at least 48px high

Add:

html,
body {
  width: 100%;
  max-width: 100%;
  overflow-x: hidden;
}

*,
*::before,
*::after {
  box-sizing: border-box;
}

img,
svg,
video,
iframe {
  max-width: 100%;
  height: auto;
}

  line-height: 1.22;

IMAGES

At 768px and below:

- Use width: 100%
- Use height: auto
- Use object-fit: cover where needed
- Avoid very tall image crops
- Use hero background-position appropriate to the subject
- If text overlays an image, use a solid semi-transparent overlay
- Do not use gradients
- If overlay text is still difficult to read, move text below the image

DROPDOWN MENUS

- Main navigation

- Be easy to tap
- Open and close without page reload
- Not require hover

ACCESSIBILITY

Ensure:

- WCAG AA contrast
- Visible focus states
- 44px minimum touch targets
- Dropdowns use buttons, not noninteractive text
- Collapsible content uses aria-expanded
- Forms retain labels
- No important content is available only by hover
- No content is hidden permanently from screen readers
