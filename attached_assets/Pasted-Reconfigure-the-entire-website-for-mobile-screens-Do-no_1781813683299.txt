MOBILE DESCRIPTION BEHAVIOR

At screen widths of 768px and below:

1. On every individual service page, show only the first two paragraphs of the service description by default.
2. Place the remaining paragraphs inside a collapsible section.
3. Add a button labeled:

“Read Full Description”

4. When opened, change the button text to:

“Show Less”

5. Do not delete or rewrite the full service description.
6. Keep all content in the HTML so it remains available for accessibility and search engines.
7. Do not collapse:
   - service-page heading
   - breadcrumb or back link
   - “What the client can expect”
   - inquiry button
   - required legal or licensing limitations
8. Any licensing, prescribed-fire, nuisance-wildlife, chemical-application, or registered-forester limitation must remain visible without requiring the user to expand the description.

OVERVIEW CARDS

- Limit each service-card description to about 2–3 lines on mobile
- Keep the service title visible
- Keep the Learn More button visible
- Do not shorten the desktop card descriptions

MOBILE PAGE LAYOUT

Across the entire website:

- Set all grids to one column
- Remove fixed widths
- Remove min-width values
- Prevent horizontal scrolling
- Set images to max-width: 100%
- Set containers to width: 100%

- Stack buttons vertically
- Make buttons full-width
- Make all form fields full-width
- Use at least 16px font size for form fields
- Keep body text at 17px minimum
- Keep line-height between 1.55 and 1.7
