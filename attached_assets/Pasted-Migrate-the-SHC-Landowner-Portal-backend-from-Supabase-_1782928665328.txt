USER ROLES
==================================================

Use Firebase custom claims for:

- super_admin
- staff_admin
- field_staff
- client_owner
- client_viewer

Custom claims must be assigned only through a trusted Cloud Function using the Firebase Admin SDK.

Do not allow the browser to assign or change roles.

Permissions:

super_admin:
- complete access
- manage staff roles
- manage clients
- manage users
- manage properties
- manage all system data
- view audit logs

staff_admin:
- manage clients, properties, projects, budgets, documents, maps, inquiries, and client access
- cannot create another super_admin

field_staff:
- update assigned field records
- upload photographs
- add monitoring notes
- cannot manage users
- cannot access unrelated financial information

client_owner:
- access assigned clients and properties
- see approved client-visible information
- send messages
- upload requested files
- respond to project actions

client_viewer:
- read-only access to approved assigned information
- may send messages if enabled

==================================================
FIREBASE SECURITY RULES
==================================================

Create strict Firestore Security Rules.

Rules must enforce:

- unauthenticated users cannot access portal data
- clients can access only records associated with their assigned clientIds
- clients can access only clientVisible records
- clients cannot access internalNotes
- clients cannot access draft documents
- clients cannot access another client
- staff permissions are based on custom claims
- only administrators can create client users
- only administrators can change roles
- audit logs are staff-only
- system settings are super-admin-only

Do not depend only on frontend filtering.

Because Firestore returns whole documents, do not store internal and client-visible sensitive fields in the same readable document when clients must access that document.

Where necessary, split records into:

- client-safe document
- internal staff-only document

Example:

projects/{projectId}
projectsInternal/{projectId}

properties/{propertyId}
propertiesInternal/{propertyId}

documents/{documentId}
documentsInternal/{documentId}

Create secure rules for each.
