- [SHC Backend Architecture](shc-backend.md) — Fully migrated to Firebase (Auth + Firestore + Storage + Cloud Functions); Netlify frontend. Supabase archived to supabase-archive/.
- [Portal/Admin Config Pattern](shc-config-pattern.md) — Browser config uses window.__SHC_FIREBASE_CONFIG__ from gitignored firebase-config.js files; generated by netlify.toml build command from FIREBASE_* env vars. Includes functionsUrl field for public Cloud Function calls.
- [Contact Form Cloud Function Pattern](shc-contact-forms.md) — Forms retain data-netlify as fallback; JS intercepts when __SHC_FIREBASE_CONFIG__.functionsUrl is present and POSTs to Firebase Cloud Function HTTPS endpoint.
- [Firebase Role Pattern](firebase-roles.md) — Roles stored in Firebase custom claims (role string + clientIds array); set only by Cloud Functions (inviteClientUser, assignRole). No role is set directly in Firestore — claims are the authority.
